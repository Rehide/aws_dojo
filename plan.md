# AWS MLA-C01 問題道場 開発計画書

## 1. プロジェクト概要

AWS Certified Machine Learning Engineer – Associate（MLA-C01）の試験対策用Webサイトを構築する。
PC・iOSの両環境で快適に利用できる過去問演習サイトを開発する。

### デザイン・コンテンツ方針

| 方針 | 詳細 |
|------|------|
| 情報設計の参考 | 学習サイト（ITパスポート過去問道場等）の「出題設定→問題表示→結果表示」という情報配置フローを参考にする |
| 独自実装 | 文言・配色・レイアウト詳細・UI部品はすべてオリジナルで設計・実装する。既存サイトのデザインをコピーしない |
| 問題コンテンツ | 問題文・選択肢・解説はすべてオリジナルで作成する。AWS公式ドキュメントを根拠として独自に執筆する |

## 2. 要件定義

### 2.1 機能要件

| # | 要件 | 詳細 |
|---|------|------|
| F1 | 問題表示 | 問題文と4つの選択肢を表示する |
| F2 | 選択肢ランダム並び替え | 出題時に選択肢の表示順をランダムに変更する |
| F3 | 正解・解説表示 | 選択肢クリック時に正誤判定と解説を即座に表示する |
| F4 | 出題モード | 「分野指定」「模擬試験形式」の2モードを用意する |
| F5 | 分野フィルタリング | MLA-C01の試験ドメインごとに問題を絞り込める |
| F6 | 問題数選択 | 10問 / 20問 / 30問 / 50問 / 100問から選択可能 |
| F7 | 問題データ | 初期リリースで100問を収録する |
| F8 | 結果表示 | 総合スコア（正解数/出題数・正答率）とドメイン別サマリを表示 |
| F9 | 弱点ハイライト | 正答率が最も低いドメインを自動で指摘する |
| F10 | 不正解のみ再挑戦 | 不正解だった問題だけで再出題できる |
| F11 | 解説ジャンプ | 結果一覧から各問題の解説へ直接遷移できる |

### 2.2 非機能要件

| # | 要件 | 詳細 |
|---|------|------|
| NF1 | レスポンシブ対応 | PC（Chrome, Safari）・iOS（Safari, Chrome）で正常表示 |
| NF2 | パフォーマンス | 静的サイトとして構築し、高速な表示を実現 |
| NF3 | デプロイ容易性 | GitHub Pages または Vercel でホスティング可能 |

## 3. MLA-C01 試験ドメイン（問題の分野分類）

AWS公式の試験ガイドに基づく4ドメイン:

| ドメイン | 内容 | 出題比率 | 初期収録目標 |
|----------|------|----------|-------------|
| Domain 1 | Data Engineering for ML（MLのためのデータエンジニアリング） | 28% | 28問 |
| Domain 2 | ML Model Development（MLモデル開発） | 26% | 26問 |
| Domain 3 | ML Model Deployment and Operationalization（MLモデルのデプロイと運用） | 22% | 22問 |
| Domain 4 | ML Solution Monitoring and Maintenance（MLソリューションの監視と保守） | 24% | 24問 |

### 4.3 問題データ構造

```json
{
  "id": 1,
  "domain": 1,
  "domainName": "Data Engineering for ML",
  "question": "問題文...",
  "choices": [
    { "id": "c1", "text": "選択肢1" },
    { "id": "c2", "text": "選択肢2" },
    { "id": "c3", "text": "選択肢3" },
    { "id": "c4", "text": "選択肢4" }
  ],
  "correctChoiceId": "c1",
  "explanation": "解説文..."
}
```

**設計意図:**
- 各選択肢は固定の `id` を持ち、正解は `correctChoiceId` で参照する
- 表示用ラベル（A / B / C / D）はシャッフル後の表示順に応じてレンダリング時に付与する
- これにより選択肢の並び替え時にも正解判定が常に正確に行える

## 6. デザイン方針

既存サイトを模倣せず、独自のビジュアルアイデンティティで設計する:

| 要素 | 方針 |
|------|------|
| カラーテーマ | ライトモード基本。ベース: スレートグレー系（#F8FAFC）、プライマリ: ディープブルー（#1E3A5F）、アクセント: ティール（#0D9488）。AWSオレンジは補助的に使用 |
| フォント | system-ui を基本とし、読みやすさを重視 |
| カードUI | 問題・設定パネルはカード形式で角丸+シャドウのモダンなデザイン |
| 選択肢ボタン | ボーダー付きカード型。ホバー時にティールのアクセント。正解時は緑系、不正解時は赤系のフィードバック |
| レスポンシブ | モバイルファースト設計。768px をブレークポイントとする |
| ヘッダー | PC: 横並びナビ、モバイル: ハンバーガーメニュー |
| 全体トーン | クリーンで落ち着いた配色。学習に集中しやすい低コントラストの背景に、要素を際立たせるアクセントカラー |

## 7. 問題データの収集方針

### 7.1 出典・作成方針

- **すべての問題文・選択肢・解説をオリジナルで作成する**（既存問題集の転載・翻訳は行わない）
- AWS公式の試験ガイド（出題範囲・ドメイン定義）を基に出題テーマを決定
- AWS公式ドキュメント、ホワイトペーパーを根拠とした正確な解説を独自に執筆
- 各ドメインの出題比率（28:26:22:24）に合わせて100問を配分

### 7.2 対象サービス・トピック（主要なもの）

| ドメイン | 主要トピック |
|----------|-------------|
| Domain 1 | SageMaker Data Wrangler, Glue, Athena, Feature Store, データ前処理, S3 |
| Domain 2 | SageMaker 組み込みアルゴリズム, ハイパーパラメータ調整, モデル評価指標 |
| Domain 3 | SageMaker Endpoints, Pipelines, Model Registry, A/Bテスト, Auto Scaling |
| Domain 4 | Model Monitor, CloudWatch, データドリフト検出, 再学習パイプライン |
